# B Trees and B+ Trees

Main idea : Reduce the height of the tree.

B-tree used for multi-level indexing of database

B+ trees, only the keys in leaf nodes have record pointers. Because every key in the tree will be in the leaf node. Leaf nodes are dense index.

Disk structure

  • Tracks
  • Sectors
  • Blocks
    • Block addres : (track #, sector #)
    • Particular byte : (track #, sector #, offset within that block)
    • ex. 512 bytes
    • Read and write to disk in blocks

Main memory (RAM)

Index

  • Dense, sparse multi-level index

Rule

  1. ceil(m/2) children

  2. Root can have min 2 children

  3. All leaf nodes at same level

  4. Creation is bottom up

One page ideally contains one node.

Maximum height of the tree is O(h).

Search, insert, delete is O(h).
